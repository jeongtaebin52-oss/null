# 우선순위 로드맵 체크리스트 (코드 기준)

규칙: 각 항목은 번호 + `[ ]` 체크박스. 완료 시 `[x]`로 변경.

1. [x] 실시간 운영 아키텍처 확정 및 배포 경로 고정  
목표: Socket.IO 기반 실시간이 끊기지 않는 운영 환경을 확정하고 문서/코드와 일치시키기.  
작업: Vercel 유지 여부 결정, Node 서버 운영이면 배포 스크립트/환경변수/포트 정책 확정, 서버 시작 경로 고정.  
관련 파일: `server.ts` `docker-compose.yml` `Dockerfile` `vercel.json`  
완료 기준: “어디에 배포하면 실시간이 100% 동작하는지” 단일 답변이 문서와 코드에 반영됨.
참고: `docs/실행_GCP_운영_아키텍처_결정.md`

2. [x] 이벤트 파이프라인 신뢰성(유실 방지)  
목표: 이벤트가 DB에 “적어도 1번” 저장되도록 설계 변경.  
작업: Redis 리스트에서 pop 후 DB 실패 시 재큐잉/백오프/데드레터 큐 도입, 배치 처리에 동시 실행 가드 추가.  
관련 파일: `src/server/eventSync.ts` `src/lib/redis.ts`  
완료 기준: DB 장애/재시작 상황에서 이벤트 유실이 발생하지 않음을 테스트로 확인.

3. [x] 이벤트 멱등성 + 스키마 검증  
목표: 중복 저장/잘못된 페이로드를 시스템적으로 차단.  
작업: `event_id` 도입, 중복 방지 인덱스 추가, 이벤트 타입/필드 검증을 런타임에서 수행.  
관련 파일: `src/app/api/pages/[pageId]/events/ingest/route.ts` `src/server/socket.ts` `prisma/schema.prisma`  
완료 기준: 동일 이벤트가 여러 번 들어와도 1회만 저장되고, 잘못된 payload는 400으로 처리됨.

4. [x] 관측성(로그/에러 추적) 표준화  
목표: 현재 “삼켜지는” 예외를 모두 추적 가능하게 변경.  
작업: 구조화 로거 도입, `catch {}` 제거 또는 최소한의 컨텍스트 로그 추가, 요청/세션 상관 ID 추가.  
관련 파일: `src/server/eventSync.ts` `src/server/socket.ts` `src/app/api/**` `src/lib/api-error.ts`  
완료 기준: 장애 발생 시 원인·경로·요청 식별이 로그로 추적 가능.

5. [x] 인메모리 의존 제거  
목표: 멀티 인스턴스/서버리스 환경에서도 동작이 동일하도록 변경.  
작업: rate limit을 Redis 기반으로 교체, publish API 임시 store를 DB/Redis로 이전.  
관련 파일: `src/lib/rate-limit.ts` `src/app/api/publish/route.ts`  
완료 기준: 인스턴스 2개 이상에서 동일한 레이트리밋/데이터 결과가 보장됨.

6. [x] 인증 플로우 정리(로그인/온보딩)  
목표: 로그인/익명 흐름이 사용자 경험 기준으로 일관되게 동작.  
작업: login → redirect 정책 통일, anon/유저 통합 전략 명확화, API 보호 기준 정리.  
관련 파일: `src/app/login/page.tsx` `src/app/signup/page.tsx` `src/lib/anon.ts` `src/lib/auth.ts` `src/app/api/me/route.ts`  
완료 기준: 로그인/익명 사용자가 같은 조건에서 일관된 페이지 접근을 보장.

7. [x] 관리자 모드 보안 강화  
목표: 운영 환경에서 관리자 기능이 안전하게 동작하도록 개선.  
작업: 키 기반 단일 인증에서 세션/역할 기반으로 전환, 감사 로그 추가.  
관련 파일: `src/lib/admin.ts` `src/lib/admin-session.ts` `src/app/api/admin/**`  
완료 기준: 관리자 기능 접근이 세션/역할 기반으로 통제되고, 주요 작업 로그가 남음.

8. [x] 데이터 모델 정합성 및 인덱스 튜닝  
목표: 분석/리플레이 쿼리의 속도와 정합성을 개선.  
작업: 이벤트/세션/통계 테이블 인덱스 재검토, 멱등성 인덱스 추가, 마이그레이션 정리.  
관련 파일: `prisma/schema.prisma` `prisma/migrations/**`  
완료 기준: 주요 분석 API 응답 시간이 목표 기준(예: p95 500ms 이하)을 충족.

9. [ ] 대형 컴포넌트 분해 및 상태 관리 정리  
목표: 유지보수 가능한 크기로 리팩터링.  
작업: UI/로직/훅 분리, 커스텀 훅/유틸 모듈화, 파일 크기 제한 적용.  
관련 파일: `src/advanced/ui/AdvancedEditor.tsx` `src/components/editor-view.tsx` `src/components/dashboard-work-view.tsx`  
완료 기준: 핵심 파일의 크기가 합리적 수준(예: 1,500~2,000줄 이하)으로 축소됨.
상태: 부분 완료 (엔트리 분리 완료, `src/advanced/ui/AdvancedEditorView.tsx` 여전히 10,000줄+)

10. [x] API 입력 검증/응답 포맷 표준화  
목표: API 전체의 입력 검증과 에러 응답을 일관화.  
작업: Zod/Valibot 도입, 모든 API가 동일한 에러 포맷을 반환하도록 정리.  
관련 파일: `src/app/api/**` `src/lib/api-error.ts`  
완료 기준: 잘못된 입력이 모두 400으로 정규화되고, 에러 포맷이 통일됨.

11. [x] 테스트 확장(E2E + 통합)  
목표: 핵심 플로우를 자동 검증.  
작업: 로그인→에디터→퍼블리시→라이브→리플레이→라이브러리 시나리오를 Playwright에 추가.  
관련 파일: `e2e/smoke.spec.ts` `tests/**` `playwright.config.ts`  
완료 기준: 핵심 시나리오 E2E가 CI에서 안정적으로 통과.

12. [x] 다크모드 도입(토큰화 포함)  
목표: 라이트/다크 테마를 일관된 디자인 토큰 기반으로 지원.  
작업: 색상/타이포/쉐도우 변수를 토큰화, `data-theme` 또는 `prefers-color-scheme` 기반 테마 전환, 토글 UI 추가.  
관련 파일: `src/app/globals.css` `src/components/**` `src/app/layout.tsx`  
완료 기준: 모든 핵심 페이지에서 다크모드가 시각적으로 일관되게 적용됨.

13. [x] 접근성/UX 마감  
목표: 키보드/스크린리더/대비 기준을 충족.  
작업: ARIA 개선, 포커스 스타일 통일, 작은 텍스트 대비 개선, 오류 메시지 UX 정리.  
관련 파일: `src/components/**` `src/app/globals.css`  
완료 기준: WCAG AA 기준 대비/포커스/키보드 네비게이션 기본 충족.

14. [x] 레포 위생 정리 및 중복 제거  
목표: 빌드 대상과 개발 대상의 경계를 명확히.  
작업: `nu11/` 처리(삭제/서브패키지/빌드 제외), 대용량 산출물 제거, 인코딩 UTF-8 통일.  
관련 파일: `nu11/**` `FULL_OUTPUT.txt` `FULL_OUTPUT_STEP3.txt`  
완료 기준: “어떤 코드가 실제 제품에 반영되는지”가 명확해지고 레포가 가벼워짐.
