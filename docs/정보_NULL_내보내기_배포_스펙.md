# NULL 배포(실제 배포) 스펙

> **의도**: 에디터에서 만든 작품을 **NULL 자체에서** 실제 웹/앱처럼 동작하게 하는 것.  
> **배포 = NULL이 호스팅**해서, NULL 도메인에 고유 URL이 생기고, 그 주소로 접속하면 작품이 **NULL 서비스 안에서** 실제 웹/앱처럼 동작.

---

## 1. 목표 (NULL 자체에서 동작)

- **Publish(공개)** 와 구분: Publish는 "NULL 안에서 24시간 LIVE로 열기 + 피드 노출 + 실시간/잔상".  
  **배포(Deploy)** 는 "**NULL 자체에서** 그 결과물을 서빙해서, **NULL 도메인 URL**로 접속하면 실제 웹/앱처럼 동작하게 하기".
- **배포 주체 = NULL**. 사용자가 "배포"하면 NULL이 그 작품을 **자체 서버에서** 서빙.  
  → 파일 다운로드해서 다른 곳에 올리는 방식이 **아니라**, **NULL에서 바로 동작하는 URL**이 생기는 것.
- 에디터에서 만든 디자인·프로토타입·변수·화면 전환을 그대로 반영한 **실제 동작하는** 화면이, **"그 사람 웹"**처럼 보이도록 URL을 준다.
- **URL 형태(목표)**: `[프로젝트슬러그].null서비스.com` 같은 **서브도메인** 형태.  
  → 방문자가 보기에 "그 사람 웹" 주소로 보이게 하고, NULL 플랫폼 안에서 호스팅·라우팅으로 처리.
- **캔버스 크기**: v1은 만든 사이즈 그대로. 반응형/앱 사이즈는 선택.

---

## 2. 범위 정리

| 구분 | 설명 |
|------|------|
| **배포 결과** | NULL 도메인에 **고유 URL** 생성. 그 URL로 접속하면 작품이 **NULL 서비스 안에서** 실제 웹/앱처럼 동작. |
| **배포 주체** | **NULL이 호스팅·서빙**. 사용자가 "배포" 버튼을 누르면 NULL이 해당 페이지를 "배포됨" 상태로 두고, 누구나 (또는 링크 아는 사람) 그 URL로 접속 가능. |
| **Publish와 차이** | Publish = 24h 한정 + 피드 노출 + 실시간/잔상. 배포 = 24h 없이(또는 별도 정책으로) NULL URL에서 계속 접근 가능. |
| **파일 내보내기** | 부가 옵션. "HTML 다운로드" 등으로 NULL 밖에서도 호스팅하고 싶을 때 사용. |

---

## 3. 동작 시나리오(목표)

1. 사용자가 에디터(심플/어드밴스드)에서 작품을 만든다.
2. **"배포"** 버튼을 누른다.
3. NULL이 그 작품을 **배포됨** 상태로 두고, **NULL 도메인 URL**을 부여한다.  
   - **목표 형태**: `[슬러그].null서비스.com` (서브도메인).  
   - 폴백/초기: `/p/[id]` 또는 `/deploy/[id]` 가능.
4. **방문자**: 그 URL로 접속하면 **웹만** 본다. NULL 로그인 없이 작품이 실제 웹/앱처럼 동작한다.
5. (선택) "배포 취소" 시 해당 URL 접근 중단. 파일 다운로드(내보내기)는 별도 옵션으로 유지 가능.

---

## 4. URL·도메인 정책

| 항목 | 내용 |
|------|------|
| **목표 URL** | `[프로젝트슬러그].null서비스.com` 형태. "그 사람 웹"이 이런 식으로 나오면 좋다는 의도. |
| **방문자 경험** | 해당 주소로 들어가면 **웹만** 보임. NULL 브랜딩/로그인 없이 작품만 서빙. |
| **구현 단계** | 서브도메인 라우팅(\*.null서비스.com)은 인프라·설정 필요. v1에서 `/p/[id]`로 먼저 제공 후, 서브도메인으로 확장 가능. |

---

## 5. 프로젝트별 대시보드

| 역할 | 경험 |
|------|------|
| **방문자** | 배포된 URL에서 **웹만** 본다. 대시보드·지표 화면 없음. |
| **소유자(제작자)** | NULL에 로그인한 뒤, **해당 프로젝트(그 사람 웹) 전용 대시보드**로 들어가서 **그 프로젝트 지표만** 본다. (방문자 수, 클릭, 잔상 등) |
| **의도** | 프로젝트마다 대시보드 하나씩. 배포 결과는 "그 사람 웹"으로 보이게, 지표는 소유자만 해당 프로젝트 대시보드에서 확인. |

---

## 6. 반응형 vs 웹/앱 분리 (선택)

- v1: NULL에서 배포된 URL은 **만든 사이즈 그대로** 서빙.
- 반응형·앱 전용 사이즈는 이후 옵션으로 추가 가능.

---

## 7. 현재 상태 및 다음 단계

- **현재**: NULL에서는 **Publish = 24h LIVE**만 있으며, "배포 = NULL 자체에서 URL로 계속 서빙"은 **구현 대상**.
- **구현 방향**:  
  - 페이지에 "배포됨" 상태 추가 (예: `deployed_at`).  
  - "배포" 시 해당 페이지를 NULL URL(예: `/p/[pageId]`)에서 **만료 없이** 접근 가능하게 함.  
  - "배포 취소" 시 접근 중단.
