# NULL 프리셋 사용성 개선 전략

> 최종 갱신: 2026-02-19  
> 핵심 질문: "비개발자가 프리셋만으로 실제 동작하는 앱을 만들 수 있는가?"  
> 현재 답: 아니오 (3/10) → 목표: 7~8/10

---

## 1. 현재 상태 진단

### 1.1 코드 검증 결과 (문서 아님, 실제 코드 기준)

| 항목 | 현재 상태 | 치명도 |
|------|----------|--------|
| 미리보기 | 텍스트 버튼만 (썸네일 없음) | 치명 |
| 시각 품질 | 회색 사각형 + 기본 텍스트 | 높음 |
| 색상 팔레트 | 5개 무채색 중심 (#FFF, #E5E7EB, #F3F4F6, #111827, #6B7280) | 높음 |
| 데이터 바인딩 | `[key]: label` 수동 매핑 (개발자만 이해) | 치명 |
| 반응형 | 고정 사이즈 (360×640, 960×600) constraints 미설정 | 높음 |
| 인앱 가이드 | 전무 | 높음 |
| 카테고리 정리 | 14개 그룹 있으나 설명 없음 | 중간 |
| 검색 | 텍스트 필터만 (태그 검색 없음) | 중간 |

### 1.2 프리셋 수량
- 자산 라이브러리: 14개 카테고리, ~90개 프리셋
- 기본 프리셋: 별도 그룹 (shapes, layouts, forms 등)
- 총 ~136개 이상

## 2. 비개발자 자유 사용을 위한 핵심 전략

### 전략 A: "보고 → 넣고 → 바꾸기" 3단계 단순화

**1단계 — 보고 (See Before Insert)**
- 프리셋 버튼에 인라인 미리보기 아이콘/색상 표시
- 각 프리셋에 한 줄 설명 추가
- 카테고리별 대표 아이콘으로 시각 구분

**2단계 — 넣고 (One-Click Insert)**
- 클릭 한 번으로 캔버스에 배치 (현재 구현됨)
- 삽입 후 자동 선택 + 자동 줌 (이미 구현)

**3단계 — 바꾸기 (Easy Customize)**
- 삽입 직후 "프리셋 가이드" 토스트 표시
- "이 프리셋은 채팅룸입니다. 메시지 영역과 입력창이 포함되어 있습니다."
- 노드 이름을 의미 있는 한글로 통일 (현재도 한글이지만 더 구체적으로)

### 전략 B: 시각 품질 혁신

현재 프리셋이 "프로토타입 와이어프레임" 수준. 이것을 "출시 가능한 UI" 수준으로 올림.

**색상 시스템 확장:**
```
기존: surface(#FFF), border(#E5E7EB), text(#111827), accent(#2563EB)
추가: gradient 시작(#6366F1→#8B5CF6), 따뜻한 accent(#F97316), 
      부드러운 배경(#F8FAFC), 카드 그림자 효과
```

**디자인 품질 기준:**
- 모든 카드: 그림자 + 둥근 모서리(12~16px)
- 버튼: 호버 대비 명확한 primary 색상
- 리스트 아이템: 충분한 패딩(12~16px), 명확한 계층
- 아바타: 그라디언트 배경으로 시각적 흥미
- 입력 필드: focus 스타일 시각화

### 전략 C: 반응형 기본 내장

- 모든 프리셋 루트에 `constraints` 설정
- 내부 스택에 `layoutSizing: { horizontal: "fill" }` 적용
- 고정 크기 대신 비율 기반 레이아웃 기본값

### 전략 D: 프리셋 설명 시스템

각 프리셋에 메타데이터 추가:
```typescript
type PresetDefinition = {
  id: string;
  label: string;
  description: string;      // 한 줄 설명
  tags: string[];            // 검색 태그
  category: PresetCategory;  // 상위 카테고리
  difficulty: 'easy' | 'medium' | 'advanced';
  hasRealApi: boolean;       // 실제 API 연동 여부
  size: { w: number; h: number };
  build: (origin) => PresetBuildResult;
};
```

## 3. 구현 우선순위

### Phase 1: 즉시 (시각+패널 UX)
1. **COLORS 확장** — 현대적 팔레트, 그라디언트, 그림자
2. **프리셋 빌더 헬퍼 품질 향상** — makeButton, makeInput, makeListItem 등 시각 개선
3. **프리셋 패널 설명 추가** — 각 프리셋 버튼에 1줄 description
4. **카테고리 아이콘** — 각 그룹에 이모지/아이콘 prefix
5. **폰트 피커** — 에디터 우측 패널 fontFamily 드롭다운

### Phase 2: 단기 (반응형+가이드)
6. **반응형 기본값** — constraints + fill sizing 모든 프리셋에 적용
7. **삽입 후 가이드 토스트** — 프리셋별 도움말 메시지
8. **태그 기반 검색** — 프리셋 tags 필드 + 검색 통합

### Phase 3: 중기 (데이터 바인딩 UX)
9. **시각적 필드 매핑 UI** — 드래그&드롭 데이터 바인딩 (장기 과제)
10. **프리셋 프리뷰 SVG** — 패널에서 미니 프리뷰 렌더링

## 4. 프리셋별 개선 포인트

### 채팅룸 (asset-chat-room, asset-chat-working)
- 현재: 리스트 아이템 2개 + 입력
- 개선: 채팅 버블 스타일(좌/우 정렬), 타임스탬프, 아바타에 그라디언트

### 할일 목록 (asset-todo-list)
- 현재: 체크박스 행 3개
- 개선: 상태별 색상(완료=초록, 대기=회색), 우선순위 표시

### 랜딩 템플릿 (asset-template-landing)
- 현재: 회색 히어로 + 카드 3개
- 개선: 그라디언트 히어로, 특징 카드에 아이콘 영역, CTA 버튼 강조

### 대시보드 (asset-template-dashboard)
- 현재: 사이드바 + 테이블
- 개선: 컬러풀 KPI 카드, 차트 영역에 시각적 패턴

## 5. 성공 지표

| 지표 | 현재 | 목표 |
|------|------|------|
| 비개발자 프리셋 활용률 | 추정 20% | 70%+ |
| 프리셋 삽입 → 수정 포기율 | 추정 높음 | 30% 이하 |
| 검색으로 프리셋 찾기 | 텍스트만 | 태그+카테고리 |

| 시각 품질 만족도 | 와이어프레임 수준 | 출시 가능 수준 |
