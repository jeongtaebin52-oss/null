## 2. §31·§32 핵심 구현 검증

- **기대**: POST `/api/pages/[pageId]/events/ingest`, CORS 허용, session_id·events 배열로 전송.
- **검증**: `src/app/api/pages/[pageId]/events/ingest/route.ts` 존재. POST로 `session_id`, `events` 수신, synthetic LiveSession 생성 후 Event 저장, `Access-Control-Allow-Origin: *` 반환. **적합.**

## 3. 주의·권장 사항

1. **Prisma generate**  
   - Segment는 **raw SQL 전용**으로 구현되어 있어 generate 없이 동작.  
   - **DailyPageStats**는 cron에서 `prisma.dailyPageStats.upsert` 사용. generate를 실행해야 해당 upsert가 동작하며, 실패 시 catch로 무시되므로 서비스 장애는 없음.

2. **Cron 설정**  
   - 예약 리포트·자동 급감 알림을 쓰려면 스케줄러에서 `GET /api/cron/daily-reports`를 매일 호출하고, `Authorization: Bearer <CRON_SECRET>` 설정 필요.

3. **외부 수집 (ingest)**  
   - 외부 페이지에서 `POST /api/pages/{pageId}/events/ingest` 호출 시 CORS `*`로 허용됨. 필요 시 도메인 제한 검토.

## 4. 결론

- 체크리스트 상 **미체크 항목 없음**, §31·§32 핵심 항목은 문서 기술과 일치하게 구현되어 있음.
- 위 주의 사항(Prisma generate, Cron, ingest CORS)만 배포·운영 시 반영하면 됨.
