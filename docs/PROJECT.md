## 1. 이게 뭘 만드는 건지

**NULL**은 **24시간 소멸성 라이브 작품 플랫폼**이다.

- **핵심 가치**: “지금 여기서만 보이는” 라이브 퍼포먼스/작품. 24시간이 지나면 자동으로 비공개(expired)된다. 영구 저장·영구 배포가 목적이 아니다.
- **흐름**:
  1. **익명/계정**으로 “페이지”(캔버스 작품)를 만든다.
  2. **편집기**에서 노드(도형·텍스트·이미지·폼 등)로 디자인하고, **게시(publish)** 하면 그 순간부터 **라이브**로 열린다.
  3. **시청자**는 `/p/:id` 또는 `/live/:pageId`에서 **실시간**으로 본다. 동시 시청자 수, 클릭 펄스, **고스트 트레이스**(마우스 궤적)가 보인다.
  4. **24시간**이 지나면 Cron이 만료 처리하고, 그 페이지는 피드/작품 보기에서 사라진다.
  5. **작가**는 **내 라이브러리**에서 현재 LIVE·drafts·히스토리(만료된 것)를 보고, 만료된 건 **재게시**할 수 있다(새 24h).
  6. **Pro 이상**은 **리플레이**(enter/leave/click/move 이벤트 재생)로 과거 세션을 볼 수 있다. Free는 리플레이 접근 시 업그레이드 안내.
- **부가**: 업보트·신고·관리자(숨김/강제만료/IP차단), 플랜(Free/Standard/Pro/Enterprise), 결제(Stripe·TODO).

**정리**: “일시적 라이브 검증/전시” 툴이다. CMS·영구 콘텐츠 저장소가 아니다.

```# 1. 사업 개요

## 1.1 서비스명

* **NULL** (임시명 유지)

## 1.2 한 문장 정의

* **NULL은 사용자가 만든 “작품(웹 페이지/공간)”을 24시간 동안 공개하고, 그 위에서 발생하는 타인의 행동(클릭/커서 이동/체류/이탈)을 실시간·잔상으로 노출해 ‘말이 아닌 행동으로’ 검증하게 만드는 범용 행동 분석 인프라(Horizontal Infrastructure)입니다.**

## 1.3 핵심 철학(제품 헌법)

* 실패란 **못 만드는 것**이 아니라 **시도하지 않는 것**
* **지루함/따분함 금지**, **불안/압박/평가감 환영**
* “위로/안전/감성”을 제공하지 않음 (단, 피싱/도배 등 악성 콘텐츠는 제재)

# 2. 문제 정의(Problem)

## 2.1 기존의 UX/반응 확인 방식의 구조적 한계

1. **설문/댓글/리뷰는 ‘말’이고 행동이 아니다**

* 사용자는 말로 포장할 수 있고, 실제 사용 흐름과 괴리가 큼

2. **개인/소규모 창작자에게 UX 리서치는 너무 비쌈**

* 테스트 설계, 모집, 운영, 분석 비용이 큼
* “즉시” 반응을 보기 어렵고 피드백 루프가 느림

3. **분석툴은 ‘내 사이트’ 내부에서만 작동**

* 남의 실험/남의 데이터가 보이지 않음
* 학습과 레퍼런스 축적이 개인 단위에 갇힘

4. **‘공개된 시험대’가 없다**

* 사람들은 비교·압박·평가를 통해 개선되는데, 기존 도구는 이 긴장을 만들지 못함

## 2.2 사용자에게 실제로 남는 결핍

* “내가 만든 걸 사람들이 **어떻게** 보는지/어디서 멈추는지/왜 떠나는지”가 보이지 않음
* 반응의 기록이 쌓이지 않음(특히 초기에는)
* 개선을 반복할 동기가 약함

---

# 3. 해결책(Solution): NULL의 핵심 구조

## 3.1 핵심 아이디어

* NULL은 **페이지를 만들고 끝**이 아니라,

  1. 만들고
  2. **24시간 공개**하고
  3. 그 위에서 벌어진 타인의 행동을
  4. **실시간+잔상(Trace)로 노출**하여
  5. “행동 기반 평가”를 받게 한다

## 3.2 킬러 UX 2개

### (A) 투명한 잔상(ghost trace)

* 앞서 방문한 사람의 커서 궤적/클릭 지점을 **투명한 잔상**으로 보여줌
* 서버 부하를 줄이기 위해 **모든 좌표 저장이 아니라 핵심 궤적 1~2개를 요약(JSON)하여 클라이언트에서 리플레이**

### (B) 실시간 라이브 클릭/커서 이벤트

* “지금 누군가 여기서 클릭하고 있다”가 보임
* 무료는 **라이브만**
* 유료는 **24시간 녹화/리플레이(영상처럼)** 제공

## 3.3 휘발성과 제약(게임 룰)

* **한 사람은 동시에 ‘공개 작품’ 1개만**
* 공개 작품은 **24시간 후 사라짐(피드/URL 접근 불가)**
* 단, 사용자의 **내 라이브러리(히스토리/초안)**에는 남아 있어 재편집/재게시 가능
* 여러 개를 동시에 올리고 싶으면 **유료 플랜**

이 구조가 만들어내는 것:

* “하나만 올릴 수 있다” → 작품의 질/긴장↑
* “24시간 뒤 사라진다” → FOMO/반복 방문↑
* “내 기록은 남는다” → 실험의 축적/개선 루프↑
* “여러 개 올리려면 유료” → 전환 논리 자연스러움

## 4.4 세그먼트 4: 전문가/직군(마케터/디자이너/창업가/학생/연구)

## 1.2 한 문장 정의

# 2. 문제 정의(Problem)

## 2.1 기존의 UX/반응 확인 방식의 구조적 한계

1. **설문/댓글/리뷰는 ‘말’이고 행동이 아니다**

* 사용자는 말로 포장할 수 있고, 실제 사용 흐름과 괴리가 큼

2. **개인/소규모 창작자에게 UX 리서치는 너무 비쌈**

* 테스트 설계, 모집, 운영, 분석 비용이 큼
* “즉시” 반응을 보기 어렵고 피드백 루프가 느림

3. **분석툴은 ‘내 사이트’ 내부에서만 작동**

* 남의 실험/남의 데이터가 보이지 않음
* 학습과 레퍼런스 축적이 개인 단위에 갇힘

4. **‘공개된 시험대’가 없다**

* 사람들은 비교·압박·평가를 통해 개선되는데, 기존 도구는 이 긴장을 만들지 못함

## 2.2 사용자에게 실제로 남는 결핍

* “내가 만든 걸 사람들이 **어떻게** 보는지/어디서 멈추는지/왜 떠나는지”가 보이지 않음
* 반응의 기록이 쌓이지 않음(특히 초기에는)
* 개선을 반복할 동기가 약함

# 3. 해결책(Solution): NULL의 핵심 구조

## 3.1 핵심 아이디어

* NULL은 **페이지를 만들고 끝**이 아니라,

  1. 만들고
  2. **24시간 공개**하고
  3. 그 위에서 벌어진 타인의 행동을
  4. **실시간+잔상(Trace)로 노출**하여
  5. “행동 기반 평가”를 받게 한다

## 3.2 킬러 UX 2개

### (A) 투명한 잔상(ghost trace)

* 앞서 방문한 사람의 커서 궤적/클릭 지점을 **투명한 잔상**으로 보여줌
* 서버 부하를 줄이기 위해 **모든 좌표 저장이 아니라 핵심 궤적 1~2개를 요약(JSON)하여 클라이언트에서 리플레이**

### (B) 실시간 라이브 클릭/커서 이벤트

* “지금 누군가 여기서 클릭하고 있다”가 보임
* 무료는 **라이브만**
* 유료는 **24시간 녹화/리플레이(영상처럼)** 제공

## 3.3 휘발성과 제약(게임 룰)

* **한 사람은 동시에 ‘공개 작품’ 1개만**
* 공개 작품은 **24시간 후 사라짐(피드/URL 접근 불가)**
* 단, 사용자의 **내 라이브러리(히스토리/초안)**에는 남아 있어 재편집/재게시 가능
* 여러 개를 동시에 올리고 싶으면 **유료 플랜**

이 구조가 만들어내는 것:

* “하나만 올릴 수 있다” → 작품의 질/긴장↑
* “24시간 뒤 사라진다” → FOMO/반복 방문↑
* “내 기록은 남는다” → 실험의 축적/개선 루프↑
* “여러 개 올리려면 유료” → 전환 논리 자연스러움

## 4.1 세그먼트 1: 일반 관객/증인(A→D)

## 4.2 세그먼트 2: 실험자(B)

* 하는 일: 페이지를 만들고 공개하여 반응 확인

## 4.3 세그먼트 3: 기록자(C)

* 하는 일: 매일 들어와 “누가 다녀갔는지(발자취)” 확인, 개선 반복
* 가치: 중독성 있는 반복 실험 루프, 축적된 히스토리

## 4.4 세그먼트 4: 전문가/직군(마케터/디자이너/창업가/학생/연구)

* 가치:

  * 마케터: CTA/카피/배치 실험
  * 디자이너: 시선 흐름/구조 학습
  * 창업가: 프로덕트 반응의 빠른 탐색
  * 학생: 포트폴리오/실험 학습
  * 연구: 익명 행동 패턴 관찰(단, 연구용 세부 설계는 확실하지 않음)

## 4.5 세그먼트 5: 기업(Enterprise)
```

```
2-5) 인기 정렬 공식(고도화 버전)

단순 추천수만 쓰면 조작됩니다. “행동 기반 + 시간 감쇠 + 조작 내성”이 필요합니다.

입력 메트릭(작품 i)

V: 방문(enter) 수

U: 유니크 세션 수(익명 id 기준; 모드 A에서는 쿠키 기반, 모드 B는 계정 기반 가능)

T: 평균 체류(초)

CTR: CTA 클릭률(주요 버튼 클릭/방문)

B: 즉시 이탈률(예: 5초 미만 leave 비율)

R: 추천 수(있다면, 그러나 가중치는 작게)

A: 어뷰징 의심 스코어(동일 IP/비정상 패턴 탐지)

시간 감쇠

age_hours에 대해 decay = exp(-age_hours / k) (k=6~12 권장)

LIVE 중에는 추가 부스트 가능(단, 과하지 않게)

기본 점수

“질”을 우선:

체류/CTR은 +

이탈은 -

예시(개념):

quality = log(1+U) 
        + 0.6*log(1+V) 
        + 0.04*T 
        + 2.5*CTR 
        - 1.8*B
        + 0.2*log(1+R)
score = quality * decay * abuse_penalty
abuse_penalty = clamp(1 - A, 0.2, 1)

어뷰징 내성(핵심)

추천(R)은 품질을 뒤집을 수 없게 “0.2 가중치”

체류(T)는 너무 긴 세션 1~2개에 휘지 않도록 winsorize(상한) 적용

동일 IP/짧은 반복 방문 등 의심 패턴은 A를 키워서 penalty
```

0) NULL 한 문장 정의(정체성)

NULL은 사용자가 만든 “작품(웹 페이지/공간)”을 배포까지 완료한 뒤, 그것을 24시간 동안 공개 시험대에 올려두고, 그 위에서 발생하는 타인의 행동(클릭/실시간 커서 이동/체류/이탈)을 실시간 + 잔상(Trace)으로 드러내 “말이 아닌 행동”으로 검증·관찰·학습하게 만드는 서비스입니다.

중요한 점은 두 개입니다.

“만들기(에디터)”가 끝이 아니라, “배포·공개·관측·개선”까지가 한 루프로 묶여 있다.

NULL은 사용자를 위로하거나 정답을 주지 않고, ‘검증 가능한 증거(행동 흔적)’를 보이게 하는 판을 만든다.

1) NULL의 철학(제품 헌법)

NULL의 철학은 기능보다 상위에 있습니다. 이 철학이 UI/룰/수익 구조까지 전부 결정합니다.

1-1. 실패 정의

실패는 못 만드는 것이 아니라 시도하지 않는 것

1-2. 감정 정책(서비스가 의도적으로 만들 감정)

절대 있어선 안 되는 것: 지루함 / 따분함

NULL이 겨냥하는 문제는 “기술 부재”가 아니라 “검증 방식의 구조적 한계”입니다.

설문/댓글/리뷰는 ‘말’
말은 포장될 수 있고, 실제 행동 흐름과 불일치가 자주 발생합니다.

UX 리서치는 개인/소규모에게 너무 비싸고 느림
테스트 설계/모집/운영/분석이 필요하고, 즉시성이 낮습니다.

기존 분석툴은 대부분 ‘내 사이트 내부’만

공개된 시험대가 없다

2-2. 사용자에게 남는 결핍

“사람들이 내 작품을 어떻게 보는지”

“어디서 멈추는지”

배포한다 (실제 동작하는 웹 결과물로)

사용자는 그 증거를 보고 수정/개선한다

리포트/분석의 역할은:

더 깊은 분석이 필요하다 → **고급 리포트(유료)**로 확장된다

10-2. 리플레이(유료 핵심)

리플레이는 “영상 파일”이 아니라,
웹에서 영상처럼 재생되는 리플레이 경험 자체가 핵심입니다.

24시간 동안의 행동을 “타임라인”으로 다시 재생

구간 탐색, 속도 조절, 주요 순간 확인

“놓친 것을 복기”해서 개선 포인트를 찾는 도구

11) 운영/제재: 공개 시험대가 되려면 반드시 필요한 레이어

공개 피드가 있고, 공유가 붙는 순간,
조작/도배/피싱은 필연적으로 유입됩니다.

그래서 NULL은 “친절”이 아니라 “룰과 제재”로 판을 유지합니다.

악성 콘텐츠 제재(숨김/정지/영구 차단)

어뷰징(조작/도배) 방지

신고 기반 운영 + (중반 이후) 자동화 고도화

이 레이어는 “부가 기능”이 아니라,
