# PROJECT.md 스펙 vs 현재 구현 — 갭 정리

> **전제**: PROJECT.md에 **대시보드·플로우·와이어프레임·로그인·사용자 화면·어드민** 등이 아주 자세히 적혀 있음.  
> **현실**: **제대로 구현된 것은 에디터뿐.** 사용자가 보는 화면·관람 화면·로그인·계정 관리·대시보드·관리/편의 기능은 **없거나 기획대로 동작하지 않음.**  
> **목적**: 스펙에 있으나 구현되지 않은 것·기획대로 안 되는 것을 한곳에 정리해, 실사용 가능 수준까지 무엇이 더 들어가야 하는지 명확히 함.  
> **기능·갭 체크는 [실행_완성_실행_리스트.md](./실행_완성_실행_리스트.md)로 통합됨.** 이 문서는 갭 **설명**(스펙/현재/갭)만 둠. 체크는 그 파일에서 함.

---

## 0. 한 줄 요약

- **활동하려면 로그인하고 쓸 수 있어야 하는데, 그 흐름이 제대로 없음.**
- **사용자 입장에서 작동하는 기능은 사실상 에디터뿐.** 관련 관리·편의 기능은 전무.
- **대시보드는 중요하다고 했는데 전혀 기획대로 안 됨.**
- **수백 가지 수준의 기능이 더 들어가야 실사용 가능.** 이 문서는 그 갭을 PROJECT.md 기준으로 정리한 것.

## 1. 로그인·계정·익명 (PROJECT.md §8, §11, 시나리오)

### 스펙 (PROJECT.md에 적힌 것)

- **익명 사용자 기반**: 첫 방문 시 anon_user_id 발급, 클라이언트 저장(localStorage). 이메일/전화 강제 없음.
- **로그인/가입**: 선택적. 유료는 결제 연동 시 선택적 이메일.
- **계정**: User(anon, optional email), Plan 저장/적용.
- **홈/피드**: "로그인/가입 없이도 구경이 바로 되고, 10초 내 이해" (1-A 목표).
- **활동(만들기·내 작품·재게시 등)**: 로그인/계정 연동 후 이용.

### 현재 구현

- 익명 발급(api/anon/init)은 있으나, **로그인 없이 쓰기 어렵고, 로그인 페이지·플로우가 기획대로 구현되지 않음.**
- auth(login/logout/signup) 라우트는 있으나 **사용자 입장에서 쓸 수 있는 로그인/가입 화면·플로우·세션 연동이 미흡.**
- **"뭔가 활동하려면 로그인하고 쓸 수 있어야 한다"가 스펙인데, 그게 제대로 없음.**

### 갭 (있어야 하는데 없음 / 안 됨)

→ **체크**: [실행_완성_실행_리스트.md](./실행_완성_실행_리스트.md) **§1** 로그인·계정·익명.

## 2. 홈/피드 — 사용자가 보는 화면 (PROJECT.md 1-A, §9 레이아웃 1)

### 스펙

- **목표**: 로그인/가입 없이도 구경이 바로 되고, 10초 내 이해.
- **상단 헤더(Sticky)**: 좌측 NULL 로고, 중앙 탭(신규/인기/시간순), 우측 [만들기](Primary), [내 작품](아이콘).
- **탭별 리스트**: 카드 단위 작품 나열.
- **카드 구성**:  
  - 헤더: 작품 제목 또는 "익명 작품 #1234", LIVE면 ● LIVE + "남은 시간 23:12:09".  
  - 썸네일(캔버스 스냅샷), 썸네일 위 "실시간 관객 n명".  
  - 메트릭 배지: 체류(avg seconds), 클릭(최근 1h 또는 전체), 이탈(%).  
  - 하단: 보러가기, 추천(👍 + 숫자), 신고(⋯ 메뉴).
- **필터(선택)**: LIVE만, 종료 임박만(1시간 이내).
- **UI 토큰**: #FFFFFF/#111111/#666666/#EAEAEA, 카드 라운드 14px, 버튼/아이콘 규칙.

### 현재 구현

- feed API(신규/인기/시간순), feed UI 컴포넌트는 있으나 **사용자 입장에서 기획한 대로 동작하지 않음.**
- **헤더·카드·메트릭·추천·신고·필터** 등이 스펙·와이어프레임대로 나오지 않거나, 연동(실시간 관객·체류·클릭·이탈)이 안 됨.
- **로그인 없이 구경**이 10초 내 이해되도록 흐름·UI가 정리되어 있지 않음.

### 갭

→ **체크**: [실행_완성_실행_리스트.md](./실행_완성_실행_리스트.md) **§2** 공통 UI, **§3** 홈/피드.

## 3. 작품 보기(관람) — 관람 화면 (PROJECT.md 1-B, §9 레이아웃 2)

### 스펙

- **목표**: "남들이 남긴 흔적 + 지금 누군가 있다"를 가장 강하게 체감.
- **상단 바(Sticky)**: ← 뒤로, 작품 제목, ● LIVE/EXPIRED, 관객 n, (공통) 실시간, (유료) 리플레이/24h 녹화 토글.
- **본문**: 중앙 작품 렌더, **그 위에 오버레이 2개**  
  - **잔상 레이어**: 과거 1~2개 궤적이 투명하게 흐르며 반복 재생, 클릭 지점 작은 원(●) 깜빡.  
  - **라이브 레이어**: 지금 접속한 타인의 커서(익명 점) 또는 "방금 클릭" 펄스. 커서는 점+짧은 꼬리, 클릭 펄스 강조.
- **우측/하단 패널**: 실시간 요약 카드 4개 — 최근 10초 클릭 수, 최근 60초 새 방문, TOP 클릭 요소, 평균 체류. "기록(유료)" CTA(무료: "지금만 볼 수 있음", 유료: "지난 24시간 리플레이 가능").

### 현재 구현

- `/p/[pageId]`, `/live/[pageId]` 페이지와 캔버스 렌더는 있으나 **고스트 트레이스·실시간이 제대로 안 보이고, 실시간으로 동작하지 않음.**
- **잔상·라이브 레이어**가 스펙대로 "실시간·반복 재생"으로 나오지 않음. (Socket·배포 환경 등 미해결.)
- **실시간 요약 카드**(최근 10초 클릭, 60초 새 방문, TOP 클릭 요소, 평균 체류)가 기획대로 동작하지 않음.

## 4. 내 라이브러리 — 대시보드 (PROJECT.md 1-D, §9 레이아웃 4)

### 스펙

- **목표**: 매일 들어오는 이유 = "발자취 확인".
- **상단**: 내 현재 LIVE 작품 카드(있으면 최상단 고정) — 남은 시간 카운트다운, 관객 수·클릭 수·체류, "작품 보기", "즉시 수정 후 재게시".
- **아래**: 나의 작품 목록 — Drafts(공개하지 않은 초안), History(과거 공개 히스토리, 무료는 제한 기간/개수, 유료는 더 길게).
- **각 항목 액션**: 열기(편집), 복제, 재게시(24h).
- **§9 "내 라이브러리"**: "내가 만든 것(초안/이전 버전)" 목록, 공개 중인 작품 1개 표시(카운트다운), 수정/재게시 버튼, 공개 히스토리(유료는 더 길게).

### 현재 구현

- library API, library-view 컴포넌트는 있으나 **대시보드가 기획한 대로 전혀 안 됨.**
- **LIVE 1개 고정·카운트다운·관객/클릭/체류·작품 보기·즉시 수정/재게시** 흐름이 스펙대로 나오지 않음.
- **Drafts/History 구분·목록·열기/복제/재게시**가 기획대로 동작하지 않거나, 사용자 입장에서 접근·이해하기 어려움.

## 5. 만들기(편집기) — 스펙 vs 구현

### 스펙 (PROJECT §9 레이아웃 3, 1-C)

- 3패널: 좌측 툴박스(Box, Text, Button, Image, Divider, Badge), 중앙 캔버스, 우측 속성 패널.
- 상단: Undo/Redo, Preview, Publish(검정), Save Draft, 제약 표시(Buttons 2/3, Text 4/6 등).
- Publish 플로우: 제목(선택), 태그(선택), 공개 시간 기본 24h, "공개하면 24시간 후 사라집니다" 고지.

### 현재 구현

- **어드밴스드 에디터**는 상대적으로 구현됨. (툴바·레이어·속성·Auto Layout·Figma 임포트 등.)
- **심플 에디터**는 구조만 있고, 제약 카운터(Buttons 2/3 등)·플랜별 제약·Save Draft·Publish 플로우가 스펙·사용자 플로우대로 완결되지 않음.
- **편집 후 배포 → 라이브 → 내 라이브러리에서 보기/재게시** 흐름이 끝까지 연결되지 않음.

## 7. 어드민 (PROJECT §3)

### 스펙

- **접근**: /admin이 아니라 /ops/<secret-slug>, secret-slug는 환경변수. 관리자 권한 없으면 404/403.
- **① 콘텐츠 관리·모니터링**: LIVE 작품 목록(작품ID, 소유자(익명ID), 남은 시간, 관객수, 클릭수, 신고수), 강제 종료/강제 삭제, 피드 제거·접근 차단, 금칙어/정책 필터, 알림(관리자 대시보드).
- **② 신고/제재**: 신고 큐(사유, 신고자, 스냅샷), 제재(일시 정지, 영구 정지, IP 차단).
- **③ 시스템 설정**: 휘발 시간(12h/24h/48h), 익명 접두사, 피드 노출 규칙(인기정렬 가중치).

### 현재 구현

- admin API(force-expire, hide, report, upvote, ip-blocks, reports 등), ops/[slug] 페이지는 있으나 **관리자 모드·대시보드가 기획대로 동작하지 않음.**
- **LIVE 목록·모니터링·신고 큐·제재·시스템 설정 UI**가 스펙대로 나오지 않음.

## 9. 기타 스펙 vs 구현 (요약)

- **24h 만료**: cron/expire·live_expires_at 있으나, 배포·Cron 환경에 따라 동작 불안정 가능. 피드/접근 차단과 연동 검증 필요.
- **이벤트 수집·샘플링·잔상 생성**: Event·GhostPoint·ghost.ts 있으나, **실시간으로 보이지 않음.** Socket·배포 전략 미해결.
- **피드 정렬/필터**: 인기 공식(quality·decay·abuse_penalty) 코드 있으나, **실제 체감·데이터 연동** 미흡.
- **유료 플랜 게이팅**: plan.ts·policy 있으나, **리플레이·슬롯·제약 완화**가 실제 기능에 반영되지 않음.
- **신고/관리**: report API·Report 모델 있으나, **신고 큐·관리자 처리 플로우**가 스펙대로 아님.
- **공통 UI**: #FFFFFF/#111111/#666666/#EAEAEA·라운드·버튼·아이콘 규칙이 **전 화면 일관 적용되지 않음.**

## 10. 결론 — "수백 가지" 갭

- PROJECT.md에는 **대시보드·플로우·와이어프레임·로그인·사용자 화면·어드민·시나리오**가 아주 자세히 적혀 있음. **그걸 꼼꼼히 확인한 뒤** 이 갭 문서를 작성함.
- **현실**: 제대로 구현된 것은 **에디터뿐.** 사용자가 보는 화면·관람 화면·로그인·계정 관리·대시보드·관리/편의 기능은 **없거나 기획대로 동작하지 않음.** 활동하려면 로그인하고 쓸 수 있어야 하는데 그 흐름도 제대로 없음.
