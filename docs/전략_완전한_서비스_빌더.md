## 1. 앱 사용자 인증 시스템

### 문제
NULL로 쇼핑몰을 만들면, 그 쇼핑몰의 "구매자"가 회원가입/로그인할 방법이 없다.
현재는 플랫폼 인증(NULL 계정)만 있고, 앱 인증(앱 사용자 계정)이 없다.

### 설계

```
모델: AppUser (페이지 스코프)
├── id: string (CUID)
├── page_id: string (FK → Page)
├── email: string
├── password_hash: string
├── display_name: string?
├── avatar_url: string?
├── role: string (default: "user")
├── metadata: JSON?
├── created_at: DateTime
└── updated_at: DateTime

모델: AppSession
├── id: string (CUID)
├── app_user_id: string (FK → AppUser)
├── token: string (unique)
├── expires_at: DateTime
└── created_at: DateTime
```

### API 엔드포인트
```
POST /api/app/[pageId]/auth/register  — 회원가입
POST /api/app/[pageId]/auth/login     — 로그인 (세션 토큰 반환)
POST /api/app/[pageId]/auth/logout    — 로그아웃
GET  /api/app/[pageId]/auth/me        — 현재 사용자 정보
PATCH /api/app/[pageId]/auth/me       — 프로필 수정
POST /api/app/[pageId]/auth/password  — 비밀번호 변경
```

### 런타임 통합
- 변수: `$app_user.id`, `$app_user.email`, `$app_user.display_name`, `$app_user.role`
- 조건: `$app_user.role == "admin"` → 관리자 전용 UI 표시/숨김
- 액션: `{ type: "appAuth", action: "login" | "register" | "logout" }`
- 프리셋: "앱 로그인", "앱 회원가입", "앱 프로필" 폼 제공
- 데이터 소유권: `AppRecord`에 `app_user_id` 필드 자동 추가 가능

## 2. 외부 API 호출

### 문제
Toss 결제, Kakao 지도, SendGrid 이메일, 소셜 로그인 등 외부 서비스 연동 불가.
NULL 내부 API만 호출할 수 있어서, 독립 서비스가 될 수 없다.

### 설계

#### 새 액션 타입
```typescript
{
  type: "apiCall";
  url: string;                    // 엔드포인트 (변수 보간 지원)
  method: "GET" | "POST" | "PUT" | "PATCH" | "DELETE";
  headers?: Record<string, string>;
  body?: Record<string, unknown>;  // 변수 보간 지원
  responseVariable?: string;       // 응답을 저장할 변수명
  errorVariable?: string;          // 에러를 저장할 변수명
  onSuccess?: PrototypeAction;     // 성공 시 후속 액션
  onError?: PrototypeAction;       // 실패 시 후속 액션
}
```

#### 프록시 API (CORS 우회 + 비밀키 보호)
```
POST /api/app/[pageId]/proxy
Body: { url, method, headers, body }
```
- 서버에서 외부 API를 대신 호출
- API 키를 클라이언트에 노출하지 않음
- 페이지별 허용 도메인 화이트리스트

#### 시크릿 관리
```
모델: AppSecret
├── page_id: string
├── key: string (예: "TOSS_SECRET_KEY")
├── value: string (암호화 저장)
└── created_at: DateTime
```
- 에디터에서 시크릿 등록: 설정 패널 → 시크릿 관리
- URL/헤더에 `{{secrets.TOSS_SECRET_KEY}}` 형태로 참조
- 클라이언트에 절대 노출되지 않음 (프록시에서만 치환)

## 3. 서버 사이드 워크플로우

### 문제
"주문이 들어오면 → 재고 차감 → 결제 처리 → 확인 이메일 발송 → 관리자 알림"
이런 비즈니스 로직을 정의할 방법이 없다.

### 설계

#### 워크플로우 모델
```
모델: AppWorkflow
├── id: string
├── page_id: string
├── name: string
├── trigger: WorkflowTrigger
├── steps: WorkflowStep[]
├── enabled: boolean
└── created_at: DateTime
```

#### 트리거 타입
```typescript
type WorkflowTrigger =
  | { type: "record_created"; collection: string }
  | { type: "record_updated"; collection: string; field?: string }
  | { type: "record_deleted"; collection: string }
  | { type: "form_submitted"; formName: string }
  | { type: "schedule"; cron: string }
  | { type: "webhook"; path: string }
  | { type: "user_registered" }
  | { type: "user_logged_in" };
```

#### 스텝 타입
```typescript
type WorkflowStep =
  | { type: "create_record"; collection: string; data: Record<string, unknown> }
  | { type: "update_record"; collection: string; id: string; data: Record<string, unknown> }
  | { type: "delete_record"; collection: string; id: string }
  | { type: "api_call"; url: string; method: string; headers?: Record<string, string>; body?: unknown }
  | { type: "send_email"; to: string; subject: string; body: string }
  | { type: "send_notification"; channel: "push" | "sms" | "discord"; message: string }
  | { type: "set_variable"; key: string; value: unknown }
  | { type: "condition"; if: ConditionExpr; then: WorkflowStep[]; else?: WorkflowStep[] }
  | { type: "loop"; collection: string; variable: string; steps: WorkflowStep[] }
  | { type: "delay"; ms: number }
  | { type: "log"; message: string };
```

#### 에디터 통합
- 우측 패널 "워크플로우" 탭
- 시각적 플로우 빌더 (트리거 → 스텝 체인)
- 각 스텝에 변수 보간 지원 (`{{record.email}}`, `{{$app_user.name}}`)

## 4. 연산/수식 변수

### 문제
`cart_total = sum(items.price * items.quantity)` 같은 동적 계산 불가.
현재 변수는 수동 설정(`setVariable`)만 가능.

### 설계

#### 수식 타입 추가
```typescript
type Variable = {
  id: string;
  name: string;
  type: "string" | "number" | "boolean" | "color";
  defaultValue: string | number | boolean;
  // 새로 추가:
  computed?: {
    formula: string;       // "sum(cart.price * cart.quantity)"
    dependencies: string[]; // ["cart.price", "cart.quantity"]
  };
};
```

#### 수식 엔진
```
지원 연산:
- 산술: +, -, *, /, %, ^
- 비교: ==, !=, >, <, >=, <=
- 논리: &&, ||, !
- 문자열: concat(), length(), substring(), replace(), upper(), lower()
- 배열: sum(), count(), avg(), min(), max(), filter(), map(), find()
- 컬렉션: collection("orders").where("status", "==", "paid").sum("total")
- 날짜: now(), formatDate(), diffDays()
- 조건: if(condition, trueValue, falseValue)
- 변수 참조: $app_user.email, $variable.cart_total
```

#### 반응성
- 의존 변수가 변경되면 computed 변수 자동 재계산
- 의존 그래프 구축 → 위상 정렬 순서로 업데이트
- 컬렉션 데이터 변경 시 관련 수식 재평가

## 5. 반응형 브레이크포인트

### 문제
고정 사이즈(360×640, 960×600)만 있어서, 실제 모바일/태블릿/데스크톱 대응 불가.

### 설계

#### 브레이크포인트 시스템
```typescript
type Breakpoint = {
  id: string;
  name: string;       // "모바일", "태블릿", "데스크톱"
  minWidth: number;   // 0, 768, 1024
  maxWidth?: number;  // 767, 1023, undefined
};

// 노드별 브레이크포인트 오버라이드
type NodeBreakpointOverride = {
  breakpointId: string;
  frame?: Partial<Frame>;
  style?: Partial<NodeStyle>;
  layout?: Partial<LayoutMode>;
  hidden?: boolean;
};
```

#### 에디터 통합
- 상단 툴바에 브레이크포인트 전환 버튼 (모바일/태블릿/데스크톱)
- 캔버스 너비가 해당 브레이크포인트로 변경
- 각 브레이크포인트에서 독립적으로 레이아웃 조정 가능
- 기본: 데스크톱 → 태블릿/모바일은 오버라이드

#### 런타임
- `window.innerWidth` 기반 현재 브레이크포인트 감지
- 노드 렌더링 시 해당 브레이크포인트 오버라이드 적용
- resize 이벤트로 실시간 전환

## 6. 구현 순서

### Phase 1: 핵심 인프라 (앱 인증 + 연산 변수)
- AppUser/AppSession Prisma 모델
- 인증 API (register/login/logout/me)
- 런타임 `$app_user` 변수 통합
- 수식 파서 + 평가 엔진
- computed 변수 반응성

→ 결과: 멀티유저 앱 + 동적 계산 가능

### Phase 2: 외부 연동 (API 호출 + 시크릿)
- `apiCall` 액션 타입 추가
- 프록시 API 엔드포인트
- AppSecret 모델 + 관리 UI
- 변수 보간 엔진 강화

→ 결과: 결제, 이메일, 지도 등 외부 서비스 연동 가능

### Phase 3: 자동화 (워크플로우)
- AppWorkflow Prisma 모델
- 워크플로우 실행 엔진
- 트리거 시스템 (레코드 변경, 폼 제출, 스케줄)
- 에디터 워크플로우 빌더 UI

→ 결과: 비즈니스 로직 자동화 가능

### Phase 4: 완성도 (반응형)
- 브레이크포인트 시스템
- 에디터 브레이크포인트 전환 UI
- 런타임 반응형 렌더링
- 프리셋 반응형 기본값

→ 결과: 모바일/태블릿/데스크톱 완전 대응
