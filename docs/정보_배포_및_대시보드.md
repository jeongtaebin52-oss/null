# 배포 및 내 작품 대시보드 안내

## 1. 배포 후 "내 작품 대시보드" 보는 법

- **URL**: 배포한 사이트 주소 + `/library`  
  예: `https://your-app.com/library` 또는 `https://null-xxx.vercel.app/library`
- **접근 방법**
  - **피드(홈)** 상단 오른쪽 **폴더 아이콘** 클릭 → "내 라이브러리"로 이동
  - 또는 주소창에 직접 `https://<배포도메인>/library` 입력
- **표시 내용** (로그인 또는 익명 초기화 후)
  - **LIVE**: 지금 공개 중인 작품 (남은 시간, 관객 수, 수정·재게시)
  - **초안**: 저장만 하고 공개하지 않은 작품
  - **과거 공개 이력**: 만료된 LIVE 이력 (재게시·복제 가능)
- 로그인하지 않으면 "내 작품만"은 익명 세션 기준으로만 보입니다. 계정 가입 후에는 해당 계정의 작품만 표시됩니다.

## 2. 실제 외부 배포 (웹/앱처럼 나가기)

이 프로젝트는 **Next.js + 커스텀 Node 서버(server.ts) + Socket.IO**를 한 프로세스에서 돌립니다.  
라이브 관람·잔상·실시간 관객 수는 **지속 연결(WebSocket)** 이 필요해서, **Vercel만으로는 실시간 기능이 동작하지 않습니다.**

### Vercel(애포)에 올리는 경우

- **가능한 것**: 정적/API 라우트, 피드·라이브러리·에디터·리플레이 페이지 등 **HTTP 요청만 쓰는 화면**
- **안 되는 것**: `/live/[pageId]`, `/p/[pageId]` 의 **실시간 관객·잔상·클릭 펄스** (Socket.IO 연결 유지 불가)
- **정리**: Vercel은 "서버리스"라서 요청 끝나면 프로세스가 사라지므로, **Socket 지속 연결을 쓸 수 없음**.  
  → 실시간 없이 "작품 보기·피드·라이브러리·에디터"만 쓰려면 Vercel 배포 가능.  
  → **실시간(LIVE 관람·잔상)까지 쓰려면 아래처럼 Node 서버를 한 번에 올려야 함.**

### 실시간까지 포함해 "진짜 외부 배포" 하는 방법

**한 번에 Node 프로세스로 서버를 띄우는 방식**이어야 합니다.

| 방식 | 설명 |
|------|------|
| **Docker** | `Dockerfile` 있음. `next build` 후 `npm run start`(= `npx tsx server.ts`) 실행. DB·Redis는 별도 매니지드 서비스 권장. |
| **Railway / Render / Fly.io** | Docker 또는 "Node 빌드 + start 커맨드"로 배포. 빌드: `npm install && npx prisma generate && npm run build`. 시작: `npm run start`. `PORT` 환경변수 지정. |
| **GCP (Compute Engine / Cloud Run)** | VM 또는 컨테이너에서 위와 동일하게 `server.ts` 실행. DB는 Cloud SQL, Redis는 Memorystore 등. |
| **AWS (ECS / EC2 / App Runner)** | EC2/ECS에서 Node 서버 실행 또는 App Runner로 컨테이너 배포. DB는 RDS, Redis는 ElastiCache 등. |

### 배포 시 꼭 할 일

1. **환경 변수**: `DATABASE_URL`(필수), `REDIS_URL`(선택, 없으면 PG 직격), `ADMIN_SECRET_SLUG`, `CRON_SECRET`, `NEXT_PUBLIC_APP_URL`(배포 URL), Stripe·Admin 등 `.env.example` 참고
2. **DB**: 프로덕션 PostgreSQL에서 `prisma migrate deploy` (또는 배포 파이프라인에 포함)
3. **Cron**: 24h 만료용 `GET /api/cron/expire` 를 매시 정각에 호출 (Vercel Cron 또는 외부 cron 서비스)
4. **실행**: `RUN_MODE=prod`(또는 `NODE_ENV=production`) + **먼저 `next build`** 한 뒤 `npm run start`

### 요약

- **내 작품 대시보드**: 배포 후 **`/library`** 로 접속 (피드 상단 폴더 아이콘으로도 이동 가능).
- **실제 웹/앱처럼 외부 배포**: Vercel만으로는 실시간 불가. **Docker 또는 Node 서버를 VM/컨테이너에서 실행**하는 방식(Railway, Render, GCP, AWS 등)으로 배포하면, 실시간(LIVE·잔상)까지 포함해 서비스 가능.

자세한 로컬 실행·환경 변수·Docker는 **루트 README.md** 참고.
