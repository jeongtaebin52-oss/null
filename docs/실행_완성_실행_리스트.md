## 32. NULL 배포(실제 배포) — NULL 자체에서 동작

> **기준 문서**: [정보_NULL_내보내기_배포_스펙.md](./정보_NULL_내보내기_배포_스펙.md).  
> **의도**: 배포 = **NULL 자체에서** 그 결과물을 서빙. NULL 도메인에 URL이 생기고, 그 주소로 접속하면 작품이 실제 웹/앱처럼 **NULL 서비스 안에서** 동작.

### 32.0 전제

- [x] **배포** 와 Publish 구분: Publish = NULL 내 24h 공개+피드. **배포 = NULL이 호스팅**해서 `/p/[pageId]` 등에서 만료 없이 접근 가능.
- [x] **결과물**: 에디터 디자인·프로토타입을 반영한 실제 동작하는 화면이 NULL URL에서 열림.

### 32.1 NULL 자체에서 배포·동작

- [x] 에디터/라이브러리에서 **"배포"** 액션: 배포 시 NULL이 해당 페이지를 배포됨 상태로 두고 **NULL 도메인 URL**(`/p/[pageId]`)에서 접근 가능.
- [x] **배포 취소** 시 해당 URL 접근 중단(deployed_at = null).
- [x] 배포된 페이지는 LIVE 만료와 무관하게 `/p/[pageId]`에서 조회·표시(WorkView).
- [x] **미배포** 페이지는 `/p/[pageId]` 접근 시 404(notFound). 배포된 페이지만 공개 URL에서 노출.

### 32.2 외부 배포 후 연동

- [x] 외부 배포 후에도 §31(리포트)·이벤트 수집 연동(트래킹 스크립트/API 등). (§31과 연동.) (POST .../events/ingest로 이벤트 전송 시 대시보드·리포트에 반영)
